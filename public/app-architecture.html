Canvas Chat · Architecture overview

A simple view of how the app works

Canvas Chat is a web app for generating and refining UI through conversation. A user describes what they want, the server asks an AI coding agent to work on an HTML canvas, and the result is shown live inside a sandboxed preview. Users can then leave feedback directly on the canvas, and that feedback becomes part of the next iteration.

1. Core architecture

User interface
Browser app

The user writes prompts, views the generated canvas, switches into feedback mode, and navigates through the app experience.

→
Preview layer
Sandboxed canvas

The generated UI is rendered as HTML inside an isolated iframe so it can be interactive without taking over the main app.

→
Application backend
Server routes

The server receives prompts and feedback submissions, coordinates the AI agent, and returns updated canvas content to the browser.

→
Generation engine
AI coding agent

The coding agent updates the HTML canvas based on the latest user request and the current state of the project session.

2. Main product loop

1

User describes a UI change

The flow starts with a natural-language request such as building a screen, refining a layout, or improving an interaction.

2

The app sends the request to the server

The server packages the current canvas context and forwards the task to the AI agent in a server-only environment.

3

The AI agent updates the canvas

The agent works on the HTML representation of the UI so the next state of the canvas stays consistent with what the user already sees.

4

The updated result is shown live

The browser receives the new HTML and refreshes the sandboxed preview so the user can immediately inspect the latest output.

3. Feedback loop

Feedback mode

The app has a dedicated feedback mode for reviewing the canvas. In this mode, feedback markers appear on top of the generated UI and the user can add or inspect comments directly in context.

View mode

Outside feedback mode, the canvas is presented cleanly as a viewing surface. Feedback controls stay hidden so the user can focus on the current design.

Floating feedback markers

Each comment is represented as a small dot positioned near the relevant part of the canvas. Selecting a dot opens a local popup for reading or editing that feedback.

Feedback becomes input to the next turn

When the user submits feedback, the app sends that context back to the server so the AI agent can incorporate the requested changes into the next canvas update.

4. Major responsibilities

Client

Owns the interactive shell: prompt entry, mode switching, preview display, feedback interactions, and general app state.

Server

Acts as the orchestration layer between the browser and the AI agent, keeping the agent runtime off the client and managing the current working session.

AI agent

Handles the actual implementation work by updating the HTML-based canvas according to the user’s request and embedded feedback context.

5. Representative building blocks

Area Role in the app
App shell Coordinates the overall experience: prompting, canvas display, feedback mode, and navigation across results.
Canvas panel Hosts the sandboxed iframe and the feedback overlay layer used for on-canvas review.
Input panel Lets the user submit new prompts and, in feedback mode, submit a set of comments for incorporation.
Feedback UI Shows floating dots and in-place popups so comments stay visually attached to the relevant elements.
Agent API Connects the frontend to the server-side coding agent and returns updated HTML for the canvas.
Session workspace Provides a persistent working context for each chat session so the agent can refine the canvas over multiple turns.

6. Key design principles

AI stays on the server

The coding agent runs only in the backend. This keeps the browser lightweight and avoids exposing server-only capabilities to client components.

The canvas is isolated

The generated UI lives in a sandboxed iframe, which makes it safer to render arbitrary HTML while still allowing rich preview behavior.

Feedback is contextual

Comments are attached directly to the canvas instead of being managed as a completely separate review workflow.

The product is iterative by design

Every prompt and feedback submission builds on the current canvas, making the experience feel more like collaborative editing than one-shot generation.

In one sentence: Canvas Chat is a conversational UI-building tool where a browser-based canvas, a server-side coding agent, and a lightweight feedback workflow work together to turn prompts into iteratively refined HTML experiences.
