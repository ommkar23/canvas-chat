01 — Overview

A live design scratchpad connected to an AI coder.

Describe UI you want built. The agent generates self-contained HTML, rendered immediately in a live canvas. Mark up the render with Figma-style feedback annotations. Submit annotated HTML back to the agent for iterative refinement — all in one surface.

02 — Goals & Non-Goals

What v1.0 ships.

✓

Streaming HTML generation

Prompt → agent streams raw HTML → canvas renders progressively in real time.

✓

Figma-style feedback layer

Click any element to pin a feedback bubble. Feedback encodes into the HTML itself as <user-feedback> elements.

✓

Version history

Every completed generation is saved. Revert to any prior canvas state non-destructively.

✓

Feedback-informed re-generation

Agent reads <user-feedback> in context HTML, applies changes, and marks resolved items.

Multi-modal input (image/file upload)
Collaborative / multi-user annotations
Export or publish generated HTML
Separate CSS / JS files — HTML only
Mobile responsiveness of the shell

Sandboxed iframe. Progressive streaming render.

iframe Sandbox

  • allow-scripts — agent-generated JS runs
  • allow-same-origin — shell can inject feedback overlay into iframe DOM
  • Agent CSS and JS cannot access the host shell

Streaming Throttle

  • HTML chunks accumulated in memory as text_delta events
  • srcdoc updated every ~100ms (capped)
  • Canvas visually builds in real time
  • Final HTML saved to version history on agent_end
05 — Feedback Mode

Figma-style annotations baked into the HTML.

Activation

  • Press F to toggle
  • Orange ● Feedback Mode badge in top bar
  • Cursor becomes crosshair inside canvas

Select & Annotate

  • Hover → element highlights with accent border
  • Click → pinned dot anchors to element corner
  • Popover card opens with textarea + Add button
  • Multiple bubbles can coexist

Bubble Controls

  • Mark resolved — close bubble
  • Mark unresolved — re-open
  • Edit — update text before submit
  • Delete — remove entirely

HTML Encoding Format

Feedback is encoded directly into the HTML — the agent reads it as context on the next prompt.

<user-feedback
  data-feedback-id="fb-001"
  data-status="unresolved"
  data-timestamp="2026-04-13T14:15:00Z"
>
  Make this button bigger and change the color to match the brand
  <div data-feedback-target>
    <button class="cta-button">Get Started</button>
  </div>
</user-feedback>

Feedback Lifecycle

StatusWho sets itMeaning
● unresolved User (on creation) Feedback added, not yet acted on by agent
✓ resolved Agent (after re-generation) Agent applied the change — bubble closes
● unresolved User (re-opened) User not satisfied — feedback re-enters next round
06 — Agent Integration

pi-coding-agent over stdio JSONL. SSE to browser.

Browser (React)
↕
HTTP / Server-Sent Events
Next.js API Route
↕
RpcClient · stdio JSONL · subprocess
pi-coding-agent (cwd = sessions/<chatId>/)
↕
Codex API (pre-configured)
Claude / LLM

Per-Session Folder

  • Each new chat → new sessions/<chatId>/ directory
  • RpcClient spawned with cwd = that directory
  • Generations saved as v1.html, v2.html, …
  • Conversation JSONL stored at ~/.pi/agent/sessions/…
  • Auth: Codex login pre-configured — nothing to set up

Streaming Events

  • agent_start → show spinner
  • message_update { text_delta } → accumulate HTML
  • turn_end → finalize render
  • agent_end → save version, write file
  • steer() / abort() — cancel mid-stream

Agent Contract — Input

Each prompt sends both the user's instruction and the full annotated canvas HTML:

{
  "prompt": "<user text>\n\nCurrent canvas HTML:\n<full iframe HTML with <user-feedback> elements>"
}

System Prompt

You are an HTML generation agent. Your sole output is a complete,
self-contained HTML document — including all CSS and JavaScript inline.
Do not output markdown, prose, or code fences.
When the input HTML contains <user-feedback> elements, read the feedback
text, apply the requested changes to the wrapped element, and set
data-status="resolved" on those elements in your output HTML.
Preserve all <user-feedback> elements unless the user explicitly asks
to remove them.

07 — Version History

In-memory ring buffer. 20 slots. Non-destructive restores.

Version Entry

{
  "version": 3,
  "timestamp": "2026-04-13T14:22:00Z",
  "prompt": "Make the hero section full width",
  "html": "<!DOCTYPE html>..."
}

Version Switcher UI

  • Compact strip or dropdown in input panel
  • Shows: version #, timestamp, truncated prompt
  • Click → read-only preview in canvas
  • Restore → set as working canvas
  • Restoring does not delete newer versions
  • Oldest versions evicted when >20 (FIFO)
